# إرشادات المساهمة في المشروع

إحنا بنرحب بكل المساهمات، سواء بتصلح Bug أو بتضيف ميزة (Feature) جديدة. عشان تحافظ على نضافة وتناسق الكود في الساحة، يا ريت تلتزم بالقواعد دي.

## أسلوب كتابة الكود (Code Style)

- **TypeScript** أساسي في كل الملفات الجديدة، مفيش تنازل عن الـ Types.
- التزم بستايل الـ **Tailwind CSS** اللي شغالين بيه في المشروع.
- استخدم الـ **Server Components** كخيار افتراضي، ومتستخدمش `"use client"` إلا للضرورة القصوى.
- خلي الـ **Components** بتاعتك صغيرة ومركزة على وظيفة واحدة محددة.

## رسائل الـ Commit

بنتبع معيار الـ [Conventional Commits](https://www.conventionalcommits.org/) عشان الـ History يفضل نضيف:
```text
feat: add leaderboard page
fix: resolve dark mode flicker on load
chore: update dependencies
docs: add contributing guidelines
refactor: simplify project card component
```

## خطوات الـ Pull Request (PR)

1. اعمل Fork للـ Repo الأصلي.
2. افتح Branch جديد للميزة اللي شغال عليها من الـ `main`.
3. خلص تعديلاتك وجربها Local وتأكد إن مفيش حاجة ضربت.
4. افتح Pull Request بعنوان ووصف يشرح إنت عملت إيه بالظبط.
5. اربط الـ PR بأي Issue ليها علاقة بالتعديل.
6. استنى الـ Review — بنحاول نرد ونراجع الكود في خلال 48 ساعة.

## مساهمات مقبولة

- ✅ تصليح الـ Bugs.
- ✅ تحسين أداء السيستم (Performance).
- ✅ ميزات جديدة بعد ما نتناقش فيها في الـ Issues الأول.
- ✅ تحسين الـ Documentation والدوكس.
- ✅ تطوير إمكانية الوصول (Accessibility).

## مساهمات مرفوضة

- ❌ تعديلات جذرية تكسر الكود (Breaking changes) من غير نقاش مسبق.
- ❌ أي PR من غير وصف واضح وتفصيلي.
- ❌ دمج تعديلات ملهاش علاقة ببعض في PR واحد.
