# الإعداد المحلي

الدليل ده هياخدك خطوة بخطوة عشان تجهز CS-Arena على جهازك المحلي وتبدأ تساهم.

## عمل Fork للمستودع

ابدأ بعمل Fork للـ Repo على GitHub، وبعدين استنسخ نسختك:
```bash
git clone https://github.com/YOUR_USERNAME/CS-Arena.git
cd CS-Arena
```

## تثبيت الاعتماديات
```bash
npm install
```

## متغيرات البيئة

هتحتاج حسابات للخدمات دي:

| الخدمة | الهدف | الباقة المجانية |
|---|---|---|
| Sanity | إدارة المحتوى وقاعدة البيانات | ✅ نعم |
| GitHub OAuth | المصادقة | ✅ نعم |
| Google OAuth | المصادقة | ✅ نعم |
| Sentry | تتبع الأخطاء | ✅ نعم |

انشئ ملف `.env.local` وحط فيه الكلام ده:
```env
NEXT_PUBLIC_SANITY_PROJECT_ID=
NEXT_PUBLIC_SANITY_DATASET=production
SANITY_API_TOKEN=

NEXTAUTH_URL=http://localhost:3000
NEXTAUTH_SECRET=

GITHUB_CLIENT_ID=
GITHUB_CLIENT_SECRET=

GOOGLE_CLIENT_ID=
GOOGLE_CLIENT_SECRET=
```

## تشغيل السيرفر
```bash
npm run dev
```

## استوديو Sanity

عشان توصل للـ CMS محلياً:
```bash
# Navigate to:
http://localhost:3000/studio
```

## تسمية الـ Branches

لما تشتغل على ميزة جديدة أو Fix، افتح Branch جديد:
```bash
# Feature
git checkout -b feat/your-feature-name

# Bug fix
git checkout -b fix/your-bug-name
```
